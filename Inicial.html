Para que serve o commit e qual a melhor forma de utilizá-lo?

O commit é uma das principais funcionalidades do sistema de controle de versão Git. Ele registra as alterações feitas nos arquivos do repositório. Ele serve para criar um ponto de referência no histórico do projeto, permitindo rastrear e comparar alterações ao longo do tempo. A melhor forma de utilizá-lo é criar commits atômicos e significativos, focados em uma única tarefa ou funcionalidade. Isso facilita a compreensão das alterações e ajuda a manter um histórico claro e organizado.

Qual a diferença entre commit e push?

O commit e o push são duas operações distintas no Git. O commit é usado para registrar alterações em um repositório local, enquanto o push é usado para enviar os commits para o repositório remoto.

Quando fazemos um commit, as alterações são gravadas apenas no repositório local. Já o push é necessário para sincronizar os commits locais com o repositório remoto, tornando as alterações disponíveis para outros colaboradores do projeto.

Dessa forma, o commit é uma operação local, que atua apenas no repositório local, enquanto o push é uma operação remota que possibilita compartilhar as alterações com outras pessoas.

O que é tag?

Uma tag no Git é uma referência estática a um ponto específico na história do projeto. Ela é usada para marcar versões importantes, como releases ou milestones, facilitando a identificação e o acesso a versões específicas do software. As tags são úteis para identificar e recuperar versões específicas do projeto de forma mais fácil. Elas geralmente são usadas quando você deseja marcar um ponto estável em seu código ou lançar uma nova versão do software para os usuários.

O que é branch?

Um branch no Git é uma ramificação independente do desenvolvimento do código-fonte. Ele permite que os desenvolvedores trabalhem em diferentes funcionalidades ou correções sem interferir no código principal. Cada branch pode ser editado, testado e fundido (merge) de volta ao branch principal (normalmente chamado de "master" ou "main") quando a funcionalidade estiver pronta ou o bug for corrigido. Branches são úteis para o desenvolvimento colaborativo, pois permitem que várias pessoas trabalhem em diferentes partes do projeto simultaneamente. Além disso, eles fornecem uma maneira de experimentar novos recursos ou correções de bugs sem modificar imediatamente a branch principal.

Como resolver possíveis conflitos no Git?

Para resolver possíveis conflitos no Git, primeiro é importante entender que conflitos ocorrem quando duas ou mais branches modificam o mesmo trecho de código. A melhor abordagem é evitar conflitos ao manter as alterações pequenas e frequentes, e ao comunicar-se com a equipe para garantir que todos estejam cientes das alterações em andamento. Se ocorrerem conflitos, a solução envolve realizar um merge ou rebase, manualmente escolhendo qual versão do código deve prevalecer. O Git sinaliza os trechos conflitantes, permitindo que o desenvolvedor revise, edite e resolva as diferenças.
